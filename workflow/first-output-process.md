# First Output Process

**Purpose:** What the LLM's first response should contain — diagnosis, story through-lines, and follow-up questions, not finished copy.

The first output should not be finished copy.

The first output is diagnosis, discovery, and story exploration.

## Required first output

When the LLM receives a worksheet, voice memo, or initial interview, respond with:

### 1. Strongest raw material

List the most promising facts, phrases, tensions, or details from the user.

### 2. Useful context

Summarize relevant context from this repository and outside research. Keep it separate from user-provided facts.

### 3. Three possible story through-lines

Each through-line should be a possible unifying narrative, not a single feature.

For each, include:

- the story in one sentence,
- which facts support it,
- who it might work for,
- what would need confirmation.

### 4. What is missing

Name the gaps that prevent stronger storytelling.

### 5. Five follow-up questions

Ask short, concrete questions.

## Good through-line format

> **A Vermont version of an old mountain solution:** This cheese may be about adapting an Alpine tradition — originally built around preserving and concentrating seasonal milk — to a Vermont farm where milk, weather, and aging conditions create a different but related challenge.

This is better than:

> The cheese is special because it is Alpine-style.

The first creates a narrative relationship. The second names a category.
