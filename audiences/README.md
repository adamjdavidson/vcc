# Audience Guides

**Purpose:** How to adapt the same story for different audiences. Each file gives the audience's posture and what they want from a story.

Different audiences need different versions of the same story. Each file in this folder gives the LLM **the audience's posture** — who they are, what they care about — and a few practical implications for how to write for them.

Do not create disconnected stories for each audience. Start with the strongest unifying narrative for the cheese, then adapt it: length, vocabulary, technical detail, sensory detail, economic detail, and practical usefulness.

## File structure

Each audience file follows the same pattern:

1. **Who they are** — the audience's posture and motivation in one tight paragraph.
2. **What they want from a story** — specifically. The thing the LLM has to deliver to land.
3. **How to write for them** — practical voice and content guidance.
4. **What to avoid** — the failure modes specific to this audience.
5. **An example pattern** when one helps.

## How the audiences think — the spine

Three core mindsets organize most of the differences. Useful to hold in mind when adapting the story:

- **Sophisticated translators** — cheesemongers. They know the cheese world and translate it for everyone else. They want depth, technical detail, and access. Don't dumb anything down.
- **Premium-curious eaters** — consumers, including farmers market shoppers. They've opted in to artisanal pricing in principle but reconfirm value with each purchase. They want to understand why this cheese is worth more, in a way they can taste, repeat, and share.
- **Sell-through thinkers** — retailers, wholesalers, and distributors (and to a lesser extent chefs and co-op buyers). Their question is *will the next link in the chain buy this from me*. They need stories that travel, language their staff can repeat, and practical mechanics that make the deal possible.

Press and food writers are the outlier — they want a *story worth telling*, not a product to sell.

## How to use these files

- Read the file matching the named audience before drafting.
- If multiple audiences are named, do not blend them. Choose the strongest fit, write that version first, and offer notes on how the story would shift for the others.
- Adapt emphasis and length to the audience. The underlying truth of the cheese stays the same.
- A good story should help the audience do something: buy, sell, serve, recommend, eat, share, write about, or remember the cheese.
