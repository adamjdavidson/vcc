# Review Checklists

**Purpose:** Pass/fail checks the LLM runs on its own draft before handing it back to the cheesemaker.

Use these checklists to evaluate your own drafts before handing them back to the cheesemaker.

Run all four when the cheesemaker has asked for finished copy. Run the relevant one mid-process when you want to check that you have not drifted.

The checklists are short on purpose. They are pass/fail rather than scaled. If a draft does not pass a check, fix the draft before showing it to the cheesemaker.

## The four checklists

- `anti-slop-checklist.md` — could this describe any cheese? does it sound like AI?
- `truth-checklist.md` — is every claim sourced and separated?
- `audience-fit-checklist.md` — does the language match who this is for?
- `final-story-checklist.md` — last pass before the cheesemaker sees it.

## How to report

When you have run a checklist on your draft, briefly tell the cheesemaker which checks the draft passed, which it failed, and what you did about the failures. Do not perform a checklist silently and ship.
